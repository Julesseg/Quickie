# Brand identity plumbed from the icon, not invented beside it

The app icon already carries a designed identity — the orbital Q: a lavender→white comet trail circling a warm gold mass over a deep purple field (`docs/brand/make-app-icon.py`). The UI, meanwhile, shipped on system defaults: an empty `AccentColor` (system blue), raw semantic badge colors, default type. We decided the app's visual identity is **derived from the icon's palette** rather than designed independently, with these laws:

- **Adaptive purple accent.** The app accent is a mid-purple (derived from the icon field) in light mode and the icon's lavender in dark mode — lavender alone fails contrast on light backgrounds. Every tint surface (highlight ring, glows, pills, toggles, Enter hint, Live Activity) inherits it.
- **Gold is the hero color, used in exactly one place.** The icon's gold names the *center of gravity*, so in the UI it marks the [[Highlighted result]] — a soft radial glow in the backdrop behind `results[0]` plus a semibold title — and appears nowhere else. Spending gold everywhere would leave nothing to mark the default action with.
- **Badge hues stay functionally distinct.** Provider badges keep one distinct hue per provider — the hue is how a row's kind is recognized at a glance — but the hues are a curated set tuned to sit with the purple accent, with no shared hues between kinds. A brand-family-only palette (all purples/golds) was rejected: it trades the palette's function for the brand. Badges render with a subtle single-hue luminosity gradient; hand-rolled shadows stay banned (ADR 0010 — depth comes from glass).
- **Rounded type on launcher chrome only.** SF Rounded on the input bar, result-row titles, favorite cards, pills, and toast — pairing with the glass/squircle language. Management pages keep system default type and native `Form` controls; they gain only backdrop parity (scroll background hidden over the shared backdrop) so the launcher aesthetic doesn't cliff at navigation. Computed/Calculator results use monospaced digits.
- **One token source, CI-guarded.** All brand constants live in a single `QuickieBrand.swift` shared by the app and widget targets (absorbing `QuickieGlyph`'s hand-copied gradients); `docs/brand/check-brand-assets.py` verifies its literals against the icon generators. Core stays color-free — Core decides *which* glyph, never how it looks. Widgets take **accents, not backdrops**: system container backgrounds are kept (widgets live on the user's wallpaper; tinted mode overrides colors anyway), with brand carried by gradient marks, accent-washed platters, and badge hues — the Entry widget remains the one full-backdrop brand tile.
